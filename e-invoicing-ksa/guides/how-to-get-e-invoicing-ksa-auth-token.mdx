---
title: "How to get E-Invoicing KSA Auth Token?"
description: "Step-by-step guide to generate an API client and auth token for the ClearTax E-Invoicing KSA API"
---

<Steps>
  <Step title="Login to your ClearTax account" icon="log-in">
    Sandbox - [https://app-sandbox.cleartax.com/middle-east/ksa/einvoicing](https://app-sandbox.cleartax.com/middle-east/ksa/einvoicing)

    Production account - [https://app.cleartax.com/middle-east/ksa/einvoicing](https://app.cleartax.com/middle-east/ksa/einvoicing/)
  </Step>

  <Step title="Go to the “Settings” page" icon="settings">
    After logging into your account, click on the ‘Settings’ option at the bottom of the left-sided menu to view all the settings available.
  </Step>

  <Step title="Go to “Integrations - API Clients” page" icon="plug">
    Among the options available in the left-sided menu ‘Settings’, click on ‘Integrations’ and select the ‘API Clients’ option in that to open the page with API clients and auth token information.
  </Step>

  <Step title="Create a new API Client" icon="key">
    Click on ‘Create API Client’ at the top to create a new one.

    a. Enter a name in the API Client Label text field which must be unique to this workspace.

    b. Click on the product, i.e. EInvoice, for which you want to create an API Client.

    c. Set the expiration time as per your need (Suggested — set to Never Expires).

    d. Click on create.
  </Step>

  <Step title="Save the generated client secret key" icon="save">
    After clicking on ‘Create’ a new API client will be created. Save the API client generated for future use in a text file or somewhere where it can be accessed easily as this will not be available later.
  </Step>

  <Step title="Generate a new access token" icon="zap">
    After generating the API client, click on ‘Generate Access Token’ to generate a new access token. Please remember to copy and save this. Once you move to another page it won't be available and you will have to generate a new one.

    Copy the Auth Token generated in the "Access Token" field and save it securely in your system. You can use this token for the request header `x-cleartax-auth-token` later in your API requests.

    Please note, you can generate as many tokens as you want (maybe for each ERP). To help you identify the tokens in the future, you can name the access tokens with labels. Make sure to keep the labels unique and identifiable.
  </Step>
</Steps>

<Callout kind="warning">
**Keep it safe!** Access token is confidential information. After creating an access token, it's important to persist it in a secure location.
</Callout>
