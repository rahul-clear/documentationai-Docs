### Invoice Type Code Object

<table><thead><tr><th width="189">Param Name</th><th width="150">Type</th><th>Validation</th><th width="431">Description</th></tr></thead><tbody><tr><td>value</td><td>String</td><td>Enum: 388, 383, 381, 386</td><td>Mandatory. A code specifying the functional type of the Invoice. Must be equal to one of the values from the subset of UN/CEFACT code list 1001, D.16B agreed for KSA electronic invoices. Allowed: 388 - Tax Invoice, 383 - Debit Note, 381 - Credit Note, 386 - Prepayment invoice (Advance invoice).</td></tr><tr><td>name</td><td>String</td><td>MinLen: 7, MaxLen: 7, regex: Numerals</td><td>Mandatory. A code of the invoice subtype and invoice transactions, structured as NNPNESBCG where NN (positions 1-2) = invoice subtype (01 tax invoice, 02 simplified tax invoice), P (3) = 3rd party invoice, N (4) = Nominal invoice, E (5) = Export invoice, S (6) = Summary invoice, B (7) = Self billed invoice (not allowed for export invoices), C (8) = Continuous supply invoice, G (9) = B2G invoice. Each flag is 0 (false) or 1 (true). For simplified tax invoices and associated notes (position 1-2 = 02), only 3rd party, nominal supply, summary, and export transactions are accepted.</td></tr><tr><td>Code (Name)</td><td>String</td><td>Allowed: 388 - Tax Invoice, 383 - Debit Note, 381 - Credit Note, 386 - Prepayment invoice</td><td>Example name values: Standard Tax Invoice (B2B) = 0100000; Standard 3rd party Invoice (B2B) = 0110000; Standard Nominal Invoice (B2B) = 0101000; Standard Export Invoice (B2B) = 0100100; Standard summary Invoice (B2B) = 0100010; Standard Self Billing Invoice (B2B) = 0100001; Continuous supply Tax Invoice (B2B) = 010000010; Simplified Tax Invoice (B2C) = 0200000; Simplified 3rd party Invoice (B2C) = 0210000; Simplified Nominal Invoice (B2C) = 0201000; Simplified summary Invoice (B2C) = 0200010; Simplified Export Invoice (B2C) = 0200100; B2G Tax Invoice = 010000001.</td></tr></tbody></table>
