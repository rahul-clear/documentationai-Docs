---
title: "Authentication"
description: "How to authenticate requests to the ClearTax E-Invoicing KSA API using a user auth token"
---

To authenticate any API request to our server, send your user authentication token in the request headers as shown below:

```
X-Cleartax-Auth-Token: {{USER_AUTH_TOKEN}}
```

This authentication token is scoped to your user account on ClearTax and can be used to access all resources linked to your user account.

Any request without a valid authentication token will be treated as a public access request. For an incorrect or missing authentication token, the response will be HTTP Status Code 401 - Unauthorized.

- **Generate** - You can generate an authentication token by logging into your ClearTax account. See [How to get E-Invoicing KSA Auth Token?](/e-invoicing-ksa/guides/how-to-get-e-invoicing-ksa-auth-token)
- **Label** - It is possible to generate multiple authentication tokens at the same time. To help you identify the tokens in the future, you can name the authentication tokens with labels.

<Callout kind="alert">
ClearTax does not store the token on its side. After generating an authentication token, if you lose or forget the token, it cannot be downloaded from the ClearTax portal again — you will need to generate a new one instead.
</Callout>

## Security

Since the authentication token is used to authenticate your account access, we highly recommend maintaining it in a secure manner.

If you suspect that the authentication token may have been compromised, you can revoke the token immediately by logging into the ClearTax account.
