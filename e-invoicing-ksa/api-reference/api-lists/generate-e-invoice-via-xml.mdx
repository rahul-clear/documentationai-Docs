Generate a valid E-Invoice with an XML file and send it to ZATCA for clearance or reporting.

### Request Method

`POST`

### Request URL

```
{{BASE_URL}}/v2/einvoices/generate-with-file
```

<Callout kind="info">
Replace `{{BASE_URL}}` with the sandbox or production base URL from the [API Reference](/e-invoicing-ksa/api-reference/api-reference).
</Callout>

<Callout kind="alert">
Rate Limit: 1000 API requests per minute
</Callout>

### Request Headers

| Parameter | Data Type | Field Validations | Description |
|---|---|---|---|
| x-cleartax-auth-token | String | Cannot be empty. | Mandatory. [User auth token](/e-invoicing-ksa/guides/how-to-get-e-invoicing-ksa-auth-token). |
| vat | String | Cannot be empty | Mandatory. VAT or Group VAT of the entity. |
| branch | String | Should be a valid branch from the user account. | Optional. Branch name (store name) added in the user account which maps to a VAT or Group VAT of the entity. |
| accept-language | String | Enum: ar, en | Optional. Language of error message from ZATCA. Default: en. |
| device-id | String | MaxLength: 36 | Mandatory. The Device ID registered with ClearTax. See [How to register a device for a VAT?](/e-invoicing-ksa/guides/how-to-register-a-device-for-a-vat) |

### Request Body

| Parameter | Data Type | Field Validations | Description |
|---|---|---|---|
| file | form-data | NA | Mandatory. |

#### Requirement of Critical Fields

Same as [Generate E-Invoice API](/e-invoicing-ksa/api-reference/api-lists/generate-e-invoice#requirement-of-critical-fields).

### API Constraints

1. This API needs to be authenticated with a valid user authentication token. If the auth token is not present or is invalid, the API will return HTTP Status Code 401.
2. Each customer is allowed to send a limited number of requests per minute to prevent server overload. If the limit is exceeded, the server will respond with a 429 Too Many Requests status code.
3. If the auth token is valid, but the user does not have the authorization to generate device ID, then the API will return HTTP Status Code 403. The user, whose auth token is used, should have an Admin role for the particular VAT used in the request.
