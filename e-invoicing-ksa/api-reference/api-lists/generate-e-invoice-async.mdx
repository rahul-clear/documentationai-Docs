---
title: "Generate E-Invoice Async"
description: "Generate a Simplified (B2C) E-Invoice synchronously and report it to ZATCA asynchronously"
---

Generate a valid E-Invoice with JSON payload synchronously and send it to ZATCA for reporting asynchronously.

<Callout kind="info">
This API can be used only for Simplified Tax Invoices and related notes (B2C).
</Callout>

### Request Method

`POST`

### Request URL

```
{{BASE_URL}}/v2/einvoices/generate/async
```

<Callout kind="info">
Replace `{{BASE_URL}}` with the sandbox or production base URL from the [API Reference](/e-invoicing-ksa/api-reference/api-reference).
</Callout>

<Callout kind="alert">
Rate Limit: 1000 API requests per minute
</Callout>

### Request Headers

| Parameter | Data Type | Field Validations | Description |
|---|---|---|---|
| x-cleartax-auth-token | String | Cannot be empty | Mandatory. User auth token. |
| vat | String | Cannot be empty | Mandatory. VAT or Group VAT of the entity. |
| branch | String | Should be a valid branch from the user account | Optional. Branch name (store name) added in the user account which maps to a VAT or Group VAT of the entity. |
| accept-language | String | Enum: ar, en | Optional. Language of error message from ZATCA. Default: en. |

### Request Body

| Parameter | Data Type | Field Validations | Description |
|---|---|---|---|
| DeviceId | String | MaxLength: 36 | Mandatory. The Device ID registered with ClearTax. |
| EInvoice | Object | Not null | Mandatory. [E-Invoice object](/e-invoicing-ksa/api-reference/resources-and-masters/e-invoice-object) |
| CustomFields | Object | NA | Optional. Custom Fields object |

**Custom Fields Object** — a custom object of key-value pairs used for custom output PDF templates or reports, e.g. `{"country_of_origin": "IN", "total_containers": "3"}`

**Requirement of Critical Fields**

| Fields | Generate E-Invoice API (via JSON) | Generate E-Invoice via XML API |
|---|---|---|
| UUID | Optional. Generated by ClearTax if not sent. | Optional. Generated by ClearTax if not sent. |
| ICV | Read Only. Ignored if sent in input. | Read Only. Ignored if sent in input. |
| PIH | Read Only. Ignored if sent in input. | Read Only. Ignored if sent in input. |
| InvoiceHash | Read Only. Ignored if sent in input. | Read Only. Ignored if sent in input. |
| Signature | Read Only. Ignored if sent in input. | Read Only. Ignored if sent in input. |
| RawQRCode | Read Only. Ignored if sent in input. | Read Only. Ignored if sent in input. |
| InvoiceXml | NA. Not available in the input schema. | NA. The XML received is enhanced before sending to ZATCA. Generating CSID (customer side) and reporting CSID (ClearTax side) differing is not supported at the moment. |

### Response Status Codes

| HTTP Status Code | Description |
|---|---|
| 202 | For successful acceptance from ClearTax |
| 400 | For any ClearTax validation error, or validation error from ZATCA |
| 401 | If the user is not authenticated for the operation |
| 403 | If the user is not authorized for the operation |
| 429 | Too Many Requests |

### Response Schema

| Parameter | Data Type | Field Validations | Description |
|---|---|---|---|
| DeviceId | String | MaxLength: 36 | Optional. Device ID used to generate invoices. |
| Status | String | Enum: GENERATED, NOT_GENERATED, GENERATION_FAILED, IN_PROGRESS | Mandatory. Same as QrCodeStatus below, kept for backward compatibility with the Phase I API. |
| QrCodeStatus | String | Enum: GENERATED, NOT_GENERATED, GENERATION_FAILED | Mandatory. Status of E-Invoice QR Code. |
| InvoiceStatus | String | Enum: PENDING, FAILED | Mandatory. Status of submission to ZATCA. |
| QRCode | String |  | Optional. Rendered QR Code image in base64 encoded PNG format. Null if validations fail. |
| RawQRCode | String |  | Optional. QR Code payload in base64 encoded TLV format. |
| InvoiceXml | String |  | Optional. The XML used for reporting. |
| UUID | String |  | Optional. UUID sent to ZATCA (generated by ClearTax if not sent in input). |
| ICV | String |  | Optional. Invoice counter value sent to ZATCA. |
| PIH | String |  | Optional. Previous hash value — invoice hash of the previous invoice reported for the same DeviceId. |
| InvoiceHash | String |  | Optional. Current invoice hash (with <code>ext:UBLExtensions</code>, <code>cac:AdditionalDocumentReference:QRCode</code> tags removed). |
| InvoiceType | String | Enum: INV, CRN, DBN | Mandatory. INV-Invoice, CRN-Credit Note, DBN-Debit Note. |
| InvoiceNumber | String | Cannot be null | Mandatory. Invoice number as per input. |
| IssueDate | String | Cannot be null | Mandatory. Issue Date as per input. |

### API Validations

API validations are the same as in Phase I, plus the following new validations for Phase II:

1. Other Seller ID is Mandatory.
2. Other Buyer ID is Mandatory.
3. Payment type code is Mandatory.
4. The document issue date must be less than or equal to the current date.
5. Buyer name is mandatory for Simplified Tax Invoice and associated credit/debit notes.
6. [BR-DEC-20] — max 2 decimals for the VAT category tax amount (BT-117).
7. [BR-DEC-13] — max 2 decimals for the Invoice total VAT amount (BT-110).
8. [BR-DEC-15] — max 2 decimals for the Invoice total VAT amount in accounting currency (BT-111).

### API Constraints

1. This API needs to be authenticated with a valid user authentication token. If the auth token is not present or is invalid, the API will return HTTP Status Code 401. If valid but unauthorized, HTTP Status Code 403.
2. Each customer is allowed to send a limited number of requests per minute to prevent server overload — exceeding the limit returns 429.
3. This async API is for simplified tax invoices only — for standard tax invoices, ZATCA generates and signs the QR code in real time. Standard invoices sent via this API will get: "Async API call is only supported for Reporting simplified tax invoices. For standard invoices use the Generate e-Invoice API."
