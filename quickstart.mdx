## Get Started with Clear

Get up and running with Clear in under 5 minutes. This guide walks you through account creation, setting up your first documentation space, adding a document, and inviting team members. Clear makes it easy to organize project docs, wikis, and guides in one collaborative space.

## Prerequisites

<Callout kind="info">

Before starting, ensure you have:
- A modern web browser (Chrome, Firefox, Safari, or Edge)
- A valid email address for account verification
- Optional: A Google or GitHub account for faster signup

No credit card or downloads required.

</Callout>

## Step-by-Step Setup

Follow these steps to create your first documentation space.

<Steps>
  <Step title="Create Your Account" icon="user-plus">

  Visit [https://clear.com/signup](https://clear.com/signup) and choose your signup method.

  <Tabs>
    <Tab title="Email" icon="mail">
      Enter your email and create a password. Clear sends a verification link—click it to activate your account.
    </Tab>
    <Tab title="Google" icon="apple">
      Click "Continue with Google" and authorize the connection. Instant access without a new password.
    </Tab>
  </Tabs>

  After signup, you'll land on the dashboard at [https://dashboard.clear.com](https://dashboard.clear.com).

  </Step>

  <Step title="Set Up Your Documentation Space" icon="folder-plus">

  From the dashboard, click **New Space**.

  1. Enter a name like "My Project Docs".
  2. Add a description: "Central hub for all project documentation."
  3. Choose visibility: Public, Private, or Workspace-only.
  4. Click **Create Space**.

  Your space appears in the sidebar. Pin it for quick access.

  </Step>

  <Step title="Add Your First Document" icon="file-plus">

  Inside your space, click **New Document**.

  Use Markdown or MDX for rich content. Here's a starter template:

  <CodeGroup tabs="Markdown,MDX">
  ```markdown
  # Welcome to My Docs

  This is your first document in Clear.

  ## Features
  - Collaborative editing
  - Version history
  - Custom domains
  ```
  ```mdx
  # Welcome to My Docs

  <Callout kind="success">This uses MDX components!</Callout>

  ## Features
  - Collaborative editing
  - Version history
  - Custom domains
  ```
  </CodeGroup>

  Click **Publish** to make it live. Share the URL instantly.

  </Step>

  <Step title="Invite Collaborators" icon="users">

  In your space settings, go to **Members** > **Invite**.

  Enter email addresses or generate a shareable link. Set roles:

  | Role       | Permissions                  |
  |------------|------------------------------|
  | Editor     | Edit docs, invite others     |
  | Viewer     | Read-only access             |
  | Admin      | Full control, billing        |

  Collaborators receive an email invite—approve to join.

  </Step>
</Steps>

## Verify Your Setup

Test your space with a simple API call to list documents (requires `{YOUR_API_KEY}` from space settings).

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X GET https://api.clear.com/v1/spaces/{SPACE_ID}/docs \
    -H "Authorization: Bearer YOUR_API_KEY"
  ```
  ```javascript
  const response = await fetch('https://api.clear.com/v1/spaces/{SPACE_ID}/docs', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  });
  const docs = await response.json();
  console.log(docs);
  ```
</Request>

Expect a JSON array of your documents.

<Expandable title="Troubleshooting Common Issues" default-open="false">

- **Signup email not received?** Check spam folder or resend from [https://clear.com/login](https://clear.com/login).
- **Document not publishing?** Ensure Markdown is valid—no unclosed code blocks.
- **Invite links broken?** Verify space permissions allow external invites.
